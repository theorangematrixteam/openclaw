## Logo Paster Workflow — Completed
- Jinay's original workflow is in `Desktop\ComfyUi_Python_new\json\logo_paster.json`
- **Jinay's rule: DON'T modify his workflows. Install missing nodes instead.**
- Original workflow uses 4 nodes: LoadImage (node 2), Image Overlay (node 3), ETN_LoadImageBase64 (node 7), ETN_SendImageWebSocket (node 8)
- **Installed 2 custom node packs:**
  - `comfyui-tooling-nodes` → provides ETN_LoadImageBase64 + ETN_SendImageWebSocket (from https://github.com/Acly/comfyui-tooling-nodes)
  - `efficiency-nodes-comfyui` → provides Image Overlay node (from https://github.com/jamesWalker55/comfyui-efficiency-nodes)
  - Both cloned to `C:\Users\openclaw.BILLION-DOLLAR-\Documents\ComfyUI\custom_nodes\`
  - efficiency-nodes needed `shapely` installed via pip in ComfyUI's Python
- **Script created:** `comfyui-scripts/logo_paster.py` — uses original workflow JSON, no modifications
- **Key technical details:**
  - ETN_LoadImageBase64 takes a base64-encoded string (not a filename) — script encodes the base image as base64
  - ETN_SendImageWebSocket sends images via WebSocket only (no file saved to disk) — script must listen on WS BEFORE queueing
  - WS binary messages have 8-byte header (event_type 2 bytes + reserved 6 bytes) before PNG/JPEG data — skip header to get image
  - Must wait for `on_open` callback before queueing, otherwise cached results finish before WS connects
  - Logo file is copied to ComfyUI input dir for LoadImage node; base image is base64-encoded
  - Image Overlay params: overlay_resize="Resize to width & heigth", resize_method="nearest-exact", opacity=0 (uses alpha mask from optional_mask input)
- **Usage:** `python logo_paster.py --base <outfit_image> --logo <logo_image> --x 3000 --y 0 --width 1024 --height 1024`
- **Output dir:** `Desktop\ComfyUi_Python_new\outputs\logo_paster\`
