# Sheet Map

## Client sheets

### Notch India
- spreadsheet title: `Notch India`
- planning tab: `Notch India`
- stronger use case: client-facing monthly plan
- weaker area: less strategic structure than Proofit

### Proofit
- spreadsheet title: `Proofit month plan`
- planning tab: `proofit`
- stronger use case: better benchmark for structured planning
- strengths:
  - clearer hooks
  - better objectives
  - stronger post angles
  - CTA included

## Internal sheet

### Core

Key tabs:

- `content tracker` — internal live queue
- `content archive` — non-active / approved / completed items moved out of the live queue
- `batches` — batch, invoice, payment, and drive tracking

## Current tracker columns

`content tracker` and `content archive` use:

1. Client
2. Month
3. Post Date
4. Platform
5. Content Type
6. Bucket
7. Topic / Hook
8. Status
9. Creation Owner
10. Draft Link
11. Approval Status
12. Scheduled Status
13. Posted Status
14. Notes

## Interpretation notes

- `Status` is the internal production state
- `Approval Status` is separate from production state
- `Scheduled Status` and `Posted Status` prevent ops blind spots
- `batches` should not replace per-content-piece tracking
