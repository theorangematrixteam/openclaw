# Content Ops Workflow

## Purpose

This workflow exists to keep recurring monthly client work visible and moving.

## Current Orange Matrix flow

1. Plan monthly content in each client's own plan sheet
2. Get plan approved by the client
3. Create content assets
4. Get content approved
5. Schedule posts
6. Track project/payment separately in `Core -> batches`

## Core operational weakness to prevent

Without the internal tracker, approved items and pending items get mixed together and the founder becomes the bottleneck.

## Correct operating split

### Client plan sheet

Use for:
- monthly strategy
- hook/topic planning
- client review
- client approval context

### Core / content tracker

Use for:
- internal active queue
- visibility into current work
- next actions
- bottleneck spotting
- creation and scheduling progress

### Core / batches

Use for:
- project-level status
- invoice tracking
- payment tracking
- drive links
- client communication shortcuts

## Queue logic

### Keep in `content tracker`

- Planned
- In Creation
- Ready for Review
- Scheduled
- anything still needing action

### Move out of live queue

Move to `content archive` when:
- internal action is finished
- item is approved and no immediate ops action remains
- item is posted/completed

If the user wants approved-but-not-created items still visible, adapt the queue policy, but default to reducing noise.

## Best-next-action logic

When reviewing the queue, prioritize in this order:

1. overdue items by post date
2. items stuck in approval
3. items in creation near deadline
4. planned items that should enter creation next
5. unscheduled approved items

## Output style

Keep outputs short and operational.

Prefer:
- what to do now
- what is blocked
- what needs approval
- what will slip if ignored
