# Content Planner

Use this skill to build monthly content plans that are actually strategic.

Do not generate filler content.
Do not plan blindly.

## Planning order

Always work in this order:

1. **Read the client knowledge file** from `clients/`
2. **Study competitor / reference brands** when links or names are available
3. **Map relevant special days, seasonal moments, and timing opportunities**
4. **Create the monthly plan with a reason behind each post**

If any of those steps are missing, the plan is weaker.

## Core rule

Every planned content piece must answer:

- why this post exists
- what strategic role it plays
- why this angle fits the brand now

If a post has no reason, it should not be in the plan.

## Client knowledge requirement

Before planning, read the relevant client file in `clients/`.

Current client knowledge files include:

- `clients/proofit.md`
- `clients/notch-india.md`

If a client file does not exist yet, create or request that knowledge before planning deeply.

## Competitor / reference analysis

Use competitor or benchmark references to understand:

- what content patterns are common in the category
- what is overused
- what visual language is working
- what positioning gaps exist
- where the client can differentiate

Do not copy competitor content.
Use competitor research to sharpen the plan.

## Special-day and timing logic

Check for:

- relevant special days
- seasonal hooks
- category-specific timing windows
- launch or collection moments
- buying-behavior timing

Only use special days when they fit the brand naturally.

Examples:

- Earth Day for a clothing brand only if sustainability or values alignment is real
- Labour Day for Proofit if tied to workmanship / build quality
- pre-monsoon for property inspection if it sharpens urgency

Do not force random calendar content.

## Planning Standard (Required Format)

Use this column structure for all client content plans:

| Column | Description |
|--------|-------------|
| Date | YYYY-MM-DD format |
| Day | Monday, Tuesday, etc. |
| Special Day | Relevant calendar hooks (Labour Day, Mother's Day, etc.) |
| Content Type | reel, carousel, post |
| Bucket | Education, Proof, Case Study, BTS, Brand, Fear |
| Status | Planned, Approved, Created, Scheduled, Posted |
| Topic / Hook | The headline/hook for the post |
| Concept and Objectives | Full concept explanation + strategic objective |
| Post Angle | How to frame the content (angle/tone) |
| CTA | Call-to-action for this post |
| Caption | Draft caption text (optional, add when creating) |

This format is mandatory for Notch India, Proofit, Orange Matrix, and all future clients.

## What good planning looks like

A good plan should:

- reduce founder thinking load
- make creation easier
- make client approval easier
- reflect brand positioning clearly
- feel purposeful, not generic

## Bucket logic

Choose buckets based on the brand, not from a fixed template.

Common buckets include:

- Product Showcase
- Proof
- Education
- Fear / objection handling
- Trends
- UGC / reality
- Brand / positioning
- Seasonal / timely
- Decision psychology

Use only what fits the client.

## Workflow

When asked to plan a month:

1. Read the client knowledge file
2. Read the client plan sheet if it exists
3. Read competitor/reference inputs if available
4. Identify relevant special-day and timing hooks
5. Draft the month row by row
6. Ensure each row has a strategic reason
7. If asked to write into Sheets, update only that client-facing sheet

## Required thinking per post

For each planned post, think through:

- what audience state this speaks to
- whether it builds awareness, trust, desire, or conversion
- whether this angle is distinct enough from generic category noise
- whether this is actually useful to create

## Output bar

Each plan should feel like:

- researched
- intentional
- brand-specific
- executable

## What to avoid

Do not:

- fill the month with random trend trash
- use special days just because they exist
- repeat the same content angle with different wording
- write vague hooks with no strategic point
- mix internal ops tracking into the client-facing sheet
