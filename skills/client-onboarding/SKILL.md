# Client Onboarding Skill

**Description**: Systematize bringing on new clients. Use when starting work with a new client, creating their folder structure, setting up their plan sheet, and generating their first month's content. Triggers on "new client," "onboard client," "start client," "add client."

---

## Onboarding Flow

### 1. Kickoff — Collect Brand Info

Get from the client:

- Brand name and niche
- Target audience (who they want to reach)
- Goals for social (awareness, sales, community, etc.)
- Competitors or brands they admire
- Preferred posting days and times
- Any must-include CTAs or messaging

This becomes their **brand brief**.

### 2. Setup — Create Their Sheet

Use `gog sheets` to create a new sheet with these tabs:

- **Plan** — content calendar with columns: Date, Day, Content Type, Status, Topic, Hook, Concept, Inspiration, Caption, CTA
- **References** — competitor pages, visual style guides, hook examples
- **Batches** — invoice/payment tracking
- **Notes** — client feedback, reminders

Name the sheet `{Client Name} — {Month} {Year}`.

Share the sheet with the client and send the link via email/DM.

### 3. First Month Plan — Generate Initial Calendar

Use marketing principles (from `memory/marketing-content-learning.md`):

- Map content pillars to their ICP (Ideal Customer Profile)
- Create a 4-week calendar with:
  - TOFU: Awareness hooks (education, trends, relatable)
  - MOFU: Consideration hooks (comparisons, case studies)
  - BOFU: Conversion hooks (offers, testimonials, demos)
- Mix formats: Reels (reach), Carousels (saves), Posts (engagement)
- Apply the 3-8-1s rule to every reel hook

Generate a draft plan and send it to the client for review/approval before scheduling.

### 4. Share — Send Welcome Package

Include:

- Their shared sheet link
- A brief welcome message outlining the process
- A note about the review/approval workflow (they approve via comments)
- Expected turnaround times

**Always draft first** — show Jinay before sending.

## Onboarding Checklist

- [ ] Collect brand brief
- [ ] Create and share their plan sheet
- [ ] Generate first month's draft content calendar
- [ ] Send welcome package with instructions
- [ ] Confirm approval process with client

## Test Case: XYZ Clothing Brand

**Brand**: XYZ (clothing brand)
**Niche**: Sustainable, ethical fashion
**Audience**: 25-40 year old conscious consumers, Gen Z, Millennials
**Goals**: Build community around sustainability, drive website traffic
**Competitors**: Reformation, Everlane, Pact
**Preferred posting**: Mon/Wed/Fri mornings

**First Month Plan**:

- Week 1: Introduce the brand mission and sustainability values
- Week 2: Educate on fabric choices, production ethics
- Week 3: Customer stories, user-generated content
- Week 4: Soft launch with limited drop

**Content Mix**:

- Reels: Behind-the-scenes of production, fabric close-ups
- Carousels: "How we make," "Fabric 101," styling tips
- Posts: Customer photos, behind-the-scenes moments

**CTAs**: "Shop the drop," "Learn more," "Join our sustainability journey"

---

## Rules

- Never start content before client approves the brief
- Use the Humanizer skill for all client docs
- Draft all messages before sending
- Share links via double confirmation
- If a client prefers email over the sheet, adapt — but default is sheet
